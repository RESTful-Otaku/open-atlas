# opencode Guide — Root Configuration

**Single source of truth** for project-wide rules. Other config files
(`AGENTS.md`, `CLAUDE.md`) reference this file. If there's a conflict,
this file wins.

## Tenets (Non-Negotiable, Priority-Ordered)

| # | Tenet | Meaning |
|---|-------|---------|
| 1 | **Correctness** | Types, invariants, exhaustive tests. If it can't be proven correct, test to exhaustion. |
| 2 | **Stability & resilience** | Graceful failure. Guard rails everywhere. Deterministic over non-deterministic. |
| 3 | **Simplicity** | Solve the problem, not more. Every abstraction carries its weight. |
| 4 | **Sensible abstractions** | Abstract when it reduces duplication or clarifies intent. YAGNI. |
| 5 | **Testing & guard rails** | Tests are the spec. Deterministic. Validation at every trust boundary. |
| 6 | **System awareness** | No change is isolated. Consider callers, implicit contracts, future refactor cost. |
| 7 | **Prove it** | Bug: test first. Optimisation: benchmark before/after. Refactor: suite passes at every step. |

## Global Directives (MUST)

1. **Read rules/ first** — Before code generation, load `rules/conventions.md`,
   `rules/architecture.md`, `rules/testing.md`, `rules/workflows.md`.
2. **Load relevant skills** — If a task matches a skill under `skills/`, load it
   before implementing.
3. **Explore before writing** — Read surrounding code, understand imports & types,
   check for existing patterns. Never invent a new one if one already exists.
4. **Type safety everywhere** — Static types: encode invariants. Dynamic languages:
   type annotations + runtime schemas. No escape hatches (`any`, `@ts-ignore`,
   `#![allow]`, `as` casts, `Object`) without documented justification.
5. **Test-first for bugs** — Write the failing test, confirm failure, then fix.
   For refactors: suite passes at every intermediate step.
6. **Verify after every change** — Run `[lint]` + `[typecheck]` + `[test]` after
   every logical unit. If any fails, fix before moving on.
7. **No guesswork** — Never invent library versions, APIs, or conventions. Check
   the codebase or ask. No exceptions.
8. **Minimize token waste** — Code over prose. No comments (unless convention
   requires). No emoji. No apologies. No "thank you". No introductions.
9. **Be self-critical** — Present trade-offs. Recommend one path. Let user decide.

## Tool-Use Policies

- **Edits**: Read before write. Use surgical `edit` over full rewrites.
- **Shell**: Dedicated tools (Read/Grep/Glob) for file ops. Bash only for
  build/test/git/package mgmt.
- **Web**: Fetch docs when needed. Prefer local sources.
- **Never hallucinate**: If uncertain, verify in codebase or ask user.

## Reporting & Analysis

When analysing, designing, or investigating:

1. Load `skills/reporting.md`.
2. Select template from `reports/templates/` for the task.
3. Populate by exploring codebase, reading files, running tools.
4. Present to user for review before proceeding.
5. Archive in `reports/YYYY-MM-DD-descriptive-name.md`.

Reports are project memory — they maintain context across sessions.

## Workflow Enforcement

1. `todowrite` must be current during multi-step sessions.
2. After every logical change: `[lint]` + `[typecheck]` + `[test]`. Blocking.
3. Commits: imperative mood, `type(scope): description` (see `rules/workflows.md`).
4. Do not commit unless explicitly asked.
