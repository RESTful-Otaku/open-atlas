# Oracle Database Skill

Loaded when the project uses Oracle Database. Supplements `rules/sql.md`
with Oracle-specific patterns.

---

## Setup

- **Connect**: `sqlplus <user>/<pass>@//<host>:1521/<service>`
- **Export**: `expdp <user>/<pass> DIRECTORY=dpump DUMPFILE=export.dmp`
- **Import**: `impdp <user>/<pass> DIRECTORY=dpump DUMPFILE=export.dmp`
- **SQLcl**: Modern CLI client with syntax highlighting and autocomplete

## Key Oracle-Specific Features

| Feature | Description |
|---------|-------------|
| PL/SQL | Procedural language for stored logic |
| Materialised views | Pre-computed views with refresh options |
| Flashback Query | Query data as of a past time (`AS OF TIMESTAMP`) |
| Partitioning | Range, list, hash, composite partitioning |
| Virtual columns | Columns computed from other columns |
| JSON | Native JSON type with JSON Path queries (v21c+) |
| Edition-based redefinition | Zero-downtime schema changes |

## Code Patterns

### Schema Design

```sql
CREATE TABLE users (
    id          NUMBER(18) GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    uuid        RAW(16) NOT NULL,
    email       VARCHAR2(255) NOT NULL,
    name        VARCHAR2(255) NOT NULL,
    metadata    JSON,
    created_at  TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT CURRENT_TIMESTAMP,
    version     NUMBER(10) NOT NULL DEFAULT 1,

    CONSTRAINT uq_users_uuid UNIQUE (uuid),
    CONSTRAINT uq_users_email UNIQUE (email)
);

CREATE INDEX idx_users_created_at ON users (created_at);
```

### Sequences (when not using IDENTITY)

```sql
CREATE SEQUENCE user_seq START WITH 1 INCREMENT BY 1 CACHE 100;
INSERT INTO users (id, ...) VALUES (user_seq.NEXTVAL, ...);
```

### JSON Queries (21c+)

```sql
SELECT u.id,
       u.name,
       u.metadata.plan.string() AS plan
FROM users u
WHERE u.metadata.plan.string() = 'premium';
```

### Flashback Query

```sql
SELECT *
FROM users AS OF TIMESTAMP (SYSTIMESTAMP - INTERVAL '1' HOUR)
WHERE id = 42;
```

## Conventions

- **Identity columns** over explicit sequences for new tables. Simpler
  syntax, less boilerplate. Use sequences only when pre-allocating IDs
  or needing cross-table ID ranges.
- **VARCHAR2 over CHAR**: `CHAR` pads values with spaces. Use `VARCHAR2`
  for variable-length strings. `CHAR` only for fixed-length codes (ISO
  country codes, status codes).
- **NUMBER over INTEGER**: Oracle's `INTEGER` is a synonym for `NUMBER(38)`.
  Specify precision explicitly: `NUMBER(18)` for IDs, `NUMBER(10,2)` for
  monetary values.
- **TIMESTAMP WITH TIME ZONE over DATE**: Oracle `DATE` has only second
  precision and no timezone. Prefer `TIMESTAMP WITH TIME ZONE` for all
  temporal data. Convert to `DATE` only for legacy compatibility.
- **Indexing**: Bitmap indexes for low-cardinality columns in data
  warehouse workloads. B-tree for OLTP. Function-based indexes for
  queries on transformed columns:
  ```sql
  CREATE INDEX idx_users_email_upper ON users (UPPER(email));
  ```
- **PL/SQL**: Use for data-intensive operations that would require
  multiple round trips. Avoid business logic in PL/SQL — keep it in
  the application layer where it's version-controlled and testable.
- **Connection pooling**: Use UCP (Universal Connection Pool),
  HikariCP (Java), or `node-oracledb` built-in pooling.
   Start with pool size = `2 × CPU cores`, monitor connection wait
   time, increase by 25% if contention is observed.
- **Partitioning**: Enable on tables > 50 GB. Range partition by date
  for time-series data. List partition for categorical data. Composite
  for very large tables.
- **Monitoring**: Query `v$session`, `v$sql`, `v$wait_class`,
  `dba_hist_sqlstat`. Set `AWR` snapshots every 60 minutes. Monitor
  `SPACE` in `dba_data_files`.
- **Backups**: RMAN for physical backups. `expdp`/`impdp` for logical.
  Archive log mode must be enabled for point-in-time recovery.
  Test RMAN restore at least quarterly.
- **No `SELECT *`** except in ad-hoc queries. Always name columns
  explicitly.
- **Naming**: `snake_case` for tables and columns. Oracle stores
  identifiers in UPPERCASE by default. Quote only if case preservation
  is needed:
  ```sql
  -- Without quotes — stored as USERS
  CREATE TABLE users (...);
  -- With quotes — stored as "Users"
  CREATE TABLE "Users" (...);  -- avoid this
  ```
